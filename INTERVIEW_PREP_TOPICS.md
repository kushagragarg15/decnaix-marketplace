## 📖 DETAILED ANSWERS TO KEY QUESTIONS

### Q: "Explain your project in 2 minutes"

**Answer Template:**
"DecnAIX is a decentralized marketplace that connects people who need computational power for AI model training with those who have it. 

**Problem:** Small companies and students often can't afford expensive GPU clusters for training ML models.

**Solution:** Our platform lets them rent computing power from individuals who have powerful machines.

**How it works:**
1. Providers register their machines with specifications
2. Tenants upload their training code (encrypted) to IPFS
3. A blockchain smart contract manages the entire transaction
4. Provider runs the training and marks it complete
5. Tenant pays via smart contract - 95% to provider, 5% platform fee
6. Tenant gets the trained model weights

**Tech Stack:** React frontend, Node.js backend, MongoDB database, Solidity smart contract, and IPFS for file storage.

**Key Innovation:** Using blockchain eliminates trust issues - payments are held in escrow and only released when work is completed."

---

### Q: "How does blockchain add value to your project?"

**Answer:**
1. **Trust & Security:** Smart contract holds payment in escrow - provider gets paid only when tenant confirms completion
2. **Transparency:** All transactions are publicly verifiable on blockchain
3. **No Middleman:** Direct peer-to-peer payments without platform holding funds
4. **Immutable Records:** Transaction history can't be altered
5. **Global Access:** Anyone with crypto wallet can participate

---

### Q: "Walk me through the payment flow"

**Answer:**
1. Tenant creates task and blockchain transaction is created
2. Provider accepts and completes training
3. Provider calls `markTaskCompleted()` on smart contract
4. Tenant verifies output and calls `payAndUnlock()` with payment
5. Smart contract automatically:
   - Deducts 5% platform fee
   - Sends 95% to provider's wallet
   - Unlocks download key for trained weights
6. All transparent and automated - no manual intervention needed

---

### Q: "How do you ensure file security?"

**Answer:**
1. **Encryption:** Files encrypted using AES-256-CBC before upload
2. **Unique Password:** Each task has unique password for decryption
3. **Decentralized Storage:** Files on IPFS, not centralized server
4. **Access Control:** Password only shared after payment confirmed
5. **Blockchain Verification:** Smart contract verifies payment before revealing password

---

### Q: "What's the biggest technical challenge you faced?"

**Answer:**
"Integrating three different technologies - traditional backend, blockchain, and IPFS - was challenging.

**Specific Challenge:** Coordinating the flow where:
- Backend uploads to IPFS
- Frontend triggers blockchain transaction
- Smart contract stores IPFS hash
- All three need to stay in sync

**Solution:** Implemented transaction-like behavior - if blockchain fails, rollback IPFS upload by unpinning the file. Used try-catch-finally blocks extensively and atomic operations where possible."

---

### Q: "How would you scale this application?"

**Answer:**
1. **Database:** Add indexes on frequently queried fields, implement caching (Redis)
2. **File Storage:** IPFS already decentralized, add CDN for faster access
3. **Backend:** Horizontal scaling with load balancer, microservices architecture
4. **Blockchain:** Move to Layer 2 solution (Polygon) for lower gas fees
5. **Queue System:** Add job queue (Bull/RabbitMQ) for heavy operations
6. **Monitoring:** Implement logging (Winston) and monitoring (Prometheus)
